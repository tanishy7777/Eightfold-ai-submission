Pipeline
Detect & Extract
Route each input to its adapter. Parses raw source into SourceRecords — partial profiles in canonical field names with per-claim {source, method, raw} provenance. A bad source returns [] + warning; run never crashes.
one adapter per source Adapter protocol robust
Normalize
Pure, deterministic functions per field type. Un-normalizable input returns None — never guessed, never invented.
phones → E.164 dates → YYYY-MM country → ISO-2 skills → canonical
Match
Cluster records that refer to the same person via connected components (union-find). Keys: normalized email (primary) then E.164 phone (fallback). No fuzzy name matching — a wrong merge is the worst outcome.
union-find transitive order-independent
Merge
Per field: scalar → trust-ranked winner. Lists → deduped union. All competing claims kept in provenance. Trust table encodes field-source affinity — contact info trusts CSV/ATS; experience trusts LinkedIn/resume; skills trust GitHub/resume.

Confidence
Score each field claim: cᵢ = trust × method_factor (direct 1.0 · regex 0.8 · inferred 0.6). Agreement → noisy-OR 1 − Π(1−cᵢ). Conflict → penalty × (1 − 0.5·disagree). overall_confidence = mean over core fields.
noisy-OR conflict penalty [0, 1] clamped
Project
Read-only pass over the canonical record. Runtime config shapes the output — rename fields, slice arrays, re-apply normalizers, set on_missing policy. Same engine, any output schema.
path DSL skills[].name emails[0] on_missing
Validate
Check projected output against config's declared type and required flags before returning. Errors surface early — never silently emit a malformed profile.
type checking required fields fail-fast
